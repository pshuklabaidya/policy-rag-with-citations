# Contributing

## Development Principles

- Keep projects reproducible.
- Use synthetic or public data only unless clear data rights exist.
- Add tests for core logic.
- Keep README claims honest.
- Document limitations.
- Document evaluation methods and known failure modes.

## Pull Request Checklist

- [ ] Code runs locally.
- [ ] Tests pass.
- [ ] Synthetic data is disclosed.
- [ ] No secrets are committed.
- [ ] README is updated.
- [ ] Evaluation notes are updated when behavior changes.
- [ ] Limitations are accurate.
