# Information Security Policy

Synthetic document for portfolio demonstration only.

## SEC-1 Passwords

Employees must use strong passwords or passphrases for all company accounts. Passwords may not be reused across personal and company systems.

## SEC-2 Multi-Factor Authentication

Multi-factor authentication is required for email, source code repositories, cloud platforms, finance systems, HR systems, and any application containing confidential data.

## SEC-3 Data Classification

Company data is classified as public, internal, confidential, or restricted. Confidential and restricted data must not be stored on personal devices or unapproved cloud services.

## SEC-4 Incident Reporting

Suspected security incidents must be reported to the security team within one hour of discovery. Incidents include lost devices, suspicious login alerts, phishing attempts, unauthorized data access, and malware warnings.

## SEC-5 Device Security

Company devices must use disk encryption, screen lock, endpoint protection, and automatic security updates. Employees may not disable security tools.

## SEC-6 Access Reviews

Managers must review employee access to sensitive systems at least quarterly. Access that is no longer required must be removed promptly.
